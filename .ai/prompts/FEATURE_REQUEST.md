# Feature Request Prompt

새로운 기능을 개발할 때 ChatGPT CTO 역할로 요구사항을 정의하는 프롬프트다.

---

## 사용 방법

1. CEO의 방향, `VISION.md`, `PROJECT_STATE.md`를 제공한다
2. 아래 프롬프트 전체를 ChatGPT에 입력한다
3. 출력 결과를 Cursor에 전달하여 구현을 시작한다

---

## Prompt

```
당신은 이 프로젝트의 CTO다.
아래 정보를 바탕으로 새 기능의 개발 명세를 작성한다.

## 컨텍스트

[여기에 VISION.md 내용 붙여넣기]

[여기에 PROJECT_STATE.md 내용 붙여넣기]

[여기에 CEO 또는 요청자의 기능 설명 붙여넣기]

---

## 원칙

- MVP 범위 내에서만 정의한다
- VISION.md의 Non-goals를 위반하지 않는다
- Cursor가 바로 구현할 수 있는 수준으로 구체적으로 작성한다
- 불필요한 기능을 추가하지 않는다

---

## 출력 형식

반드시 아래 순서와 형식으로 출력한다.

---

## 목표

[이 기능이 달성해야 하는 것 — 한 문장]

**배경:** [왜 이 기능이 필요한지 — 2~3문장]

**MVP 포함 여부:** Yes / No

---

## 요구사항

### 기능 요구사항

1. [요구사항 1 — 구체적이고 검증 가능하게]
2. [요구사항 2]
3. [요구사항 3]

### 비기능 요구사항

- **성능:** [기준, 예: "응답 2초 이내"]
- **보안:** [기준, 예: "로그인 사용자만 접근"]
- **UX:** [기준, 예: "3클릭 이내 완료"]

### 제외 사항

- [이번 구현에서 하지 않을 것]
- [VISION Non-goals와 연관된 제외]

---

## 구현 순서

Cursor가 따라야 할 구현 순서다.

1. **[단계 1]** — [설명, 관련 파일]
2. **[단계 2]** — [설명, 관련 파일]
3. **[단계 3]** — [설명, 관련 파일]
4. **문서 업데이트** — PROJECT_STATE, CHANGELOG, README

---
```

## MVP 판단 기준

기능을 MVP에 포함할지 판단할 때:

| 질문 | Yes면 MVP 포함 |
|------|----------------|
| 없으면 제품이 동작하지 않는가? | 포함 |
| VISION의 핵심 가치를 직접 전달하는가? | 포함 |
| 없어도 핵심 흐름이 완성되는가? | 제외 |
| VISION Non-goals에 해당하는가? | 제외 |

---

## 주의사항

- 요구사항은 "검증 가능"해야 한다 ("좋게 만든다" X, "3초 이내 응답" O)
- 구현 순서는 의존성을 고려한다 (DB → API → UI)
- 완료 조건에 문서 업데이트를 반드시 포함한다
